# index-bbsport-guide

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存储，不针对任何特定域名或网站，仅作为 HTML 资料的集合与索引。

所有内容均以原始 HTML 形式存放，方便通过 GitHub Pages 或其他静态托管服务直接访问。

## 目录说明

```
/
├── index.html          # 主入口页面（可选）
├── pages/              # 独立 HTML 页面存放目录
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：存放所有独立 HTML 页面文件，每个文件为一个独立页面。
- **assets/**：存放页面所需的公共资源，如样式表、脚本、图片等。
- **index.html**：可选的首页，用于导航或展示页面列表。

## 页面归档说明

- 每个 HTML 页面应为自包含的独立文件，可脱离仓库环境直接运行。
- 页面之间不强制关联，但建议保持统一的资源引用路径。
- 页面内容不涉及具体域名、网址或商业推广，仅用于技术展示或存档。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并更新 `index.html`（如有）中的链接。
- 公共资源尽量统一放置于 `assets/` 目录，避免重复。
- 提交前请确保页面无外部依赖，或已明确标注依赖来源。
- 本仓库仅为归档用途，不提供服务保证或内容更新承诺。

## 使用方式

1. 克隆仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件，或通过本地服务器预览。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能直接访问 `index.html`。
